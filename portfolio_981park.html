프로젝트 개요

배경

제주도 중력 레이싱 테마파크 9.81파크의 O2O 앱을 PO로서 두 차례 기획. 1.0은 0→1 앱 구축, 2.0은 사용자 데이터와 현장 리서치를 기반으로 한 전면 리뉴얼. 오프라인 체험을 디지털 경험으로 연결하고, 다시 오프라인 재방문으로 이어지는 루프를 설계하는 것이 핵심 과제였다.

핵심 미션

레이싱 기록 → 공유 → 배틀 → 재방문으로 이어지는 O2O 참여 루프 설계. 배틀 모드(온라인 게임 로비 방식), 랭킹, AR 가챠 이벤트 기획. 현장 크루 인터뷰를 통해 운영 페인포인트 발굴 및 앱으로 해결. B급 유머 톤 앤 매너의 UX 라이팅으로 파크 정체성과 앱 경험 일체화.

데이터가 드러낸 구조적 문제
30%
전체 유저 중 만렙 도달 비율 (GA 분석)

MAU 24만 중 약 7만 명 이상이 이미 게임 구조의 끝을 본 상태였다. 단순히 콘텐츠 업데이트 속도 문제로 봤는데, 만렙 이후에도 잔류하는 유저들이 히든맵에 집중된다는 걸 발견하면서 해석이 바뀌었다. 사용자는 끝이 있는 구조보다 예측 불가능한 보상에 더 반응하고 있었다.

체중 감량 앱의 이탈 원인은 "의지 부족"이 아니라 "구조의 문제"다.
체중 1kg 감량보다, "이번 주 샐러드 3번 먹었다"는 작은 성공의 누적이 커뮤니티 응원과 결합할 때 더 긴 retention을 만든다.

문제 해결
01

1회성 체험 → 재방문 동기 구조 설계

상황

테마파크 특성상 1회 방문 후 재방문 동기가 약했다. 앱이 체험 후 아무 역할도 하지 못하는 상태.

분석

레이싱 기록은 앱에만 존재. 배틀·랭킹·AR 가챠를 앱에서만 가능하게 하면 체험 후 앱 접속이 자연스럽게 유발된다.

결과

O2O 이벤트 참여율 70%, 영상 공유 2.6배 증가. 코로나 기간에도 연 30만 방문객 유지.

02

배틀 진입 마찰 → 온라인 게임 로비 구조 도입

상황

배틀 모드 기획 시 "방 만들기"의 진입 마찰이 높아 사용률이 낮을 것으로 예상됨.

분석

온라인 게임처럼 방 이름 자동 생성("커피 내기", "한 판 승부")으로 결정 피로 제거. A/B 테스트로 진입 포인트 최적화.

결과

배틀 참여율 목표 초과 달성. 가벼운 경쟁 구조가 소셜 공유로 자연스럽게 연결됨.

03

크루 운영 부하 → 앱으로 현장 업무 분산

상황

제주 현장 거주 리서치 중 크루들의 반복 업무 과부하 발견. 예약 확인, 대기 안내, 안전 교육 등이 모두 수동으로 처리됨.

분석

앱으로 예약 테이블, 대기 시간 표시, 안전 교육 영상을 제공하면 크루 업무가 줄고 사용자 경험도 개선된다.

결과

크루 업무 감소 + 대기 중 앱 체류 시간 증가. 현장 인터뷰에서 발굴한 인사이트가 직접 기능으로 구현된 케이스.

Lesson Learned

✓ 유지해야 할 것

현장 거주 리서치 — 데이터로 못 보이는 것은 직접 봐야 한다
오프라인 → 앱 → 재방문 루프 설계 — O2O의 본질은 연결이 아니라 순환
B급 유머 톤 앤 매너 — 서비스 정체성과 UX 라이팅의 일체화
크루 등 운영자 페인포인트를 앱으로 해결 — 사용자만이 아닌 운영자도 고객

⚠ 보완해야 할 것

배틀 기능 A/B 테스트를 더 빠르게 진행했어야 했다
코로나 같은 외부 변수에 대한 앱 전략 시나리오를 미리 준비했더라면
오프라인 이벤트와 앱 연동 데이터 트래킹 체계를 더 정교하게 구축했으면
"가장 잘한 기획 결정을 하나만 꼽으라면, 9.81파크다"

데이터가 없었다. 제주도로 직접 갔다. 크루를 인터뷰했다. 방문객 동선을 직접 따라다녔다.

그 과정에서 앱이 해결해야 할 것과 해결할 수 없는 것을 구분했고, 오프라인 체험이 앱을 필요로 하게 만드는 구조를 설계했다.
기획자가 현장을 떠나면 안 된다는 걸 이 프로젝트에서 배웠다.
