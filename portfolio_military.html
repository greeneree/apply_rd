프로젝트 개요

배경

500만 현역 장병 대상 정신건강 앱 0→1 기획 PM. 한양대 디지털헬스케어센터와 협업, 우울·불안·스트레스 검사 → 맞춤 콘텐츠 → GPT-4 기반 자살 위험 감지 → ML 상담 챗봇으로 이어지는 AI 헬스케어 파이프라인 설계. Azure Cloud SaaS 멀티테넌트 플랫폼 구조 기획.

핵심 미션

초기 타겟은 사병이었으나 데이터를 보니 간부 비율이 60% 이상 — 핸드폰 사용 시간이 더 길기 때문이었다. 타겟 재정의 후 콘텐츠 톤 조정. 밀리패스·나라사랑 제휴 채널 전략으로 모객 3개월 조기 달성. 프로모션 종료 후에도 일평균 120건 콘텐츠 이용 유지 — retention 구조 검증.

데이터가 드러낸 타겟의 진실
60%+
전체 회원 중 간부 비율 (일기·채팅 이용 시간 분석)

사병을 위해 만들었는데 간부가 더 많이 썼다. 일기 작성 시간과 채팅 이용 시간을 분석하니 근무 시간 또는 늦은 저녁까지 서비스를 이용하는 패턴이 확인됐다. 일반 병사는 폰 사용 시간이 제한적이지만, 간부는 시간에 구애받지 않았다. 데이터가 맞았다. 내 가설이 틀렸다.

왜 GPT-4와 ML을 분리했나: GPT-4는 자살 위험 감지에, ML은 상담 챗봇에 독립 운영했다. 하나가 장애나도 다른 하나는 동작해야 했기 때문. 두 모듈이 상호 의존하면 동시 실패 위험이 생긴다 — 헬스케어 서비스에서 이건 허용할 수 없었다.
문제 해결
01

타겟 가설 붕괴 → 데이터 기반 피벗

상황

사병 타겟으로 기획했으나 실제 이용자 60% 이상이 간부. 콘텐츠 톤과 채널 전략이 맞지 않는 상태.

분석

간부는 핸드폰 사용 시간이 길고, 스트레스 관리에 대한 인식도 높다. 타겟을 바꾸면 콘텐츠 톤과 채널 전략이 동시에 바뀐다.

결과

타겟 재정의 + 콘텐츠 톤 조정. 목표 1만명을 3개월 만에 초과, 1.5만명 달성.

02

초기 모객 → 밀리패스 제휴 채널 전략

상황

군 특수 환경상 일반 마케팅 채널 접근 불가. 군 내부 채널을 통한 모객이 필요했다.

분석

밀리패스·나라사랑 제휴 채널 + 경품 프로모션 설계로 초기 모객 가속화. 단, 프로모션 종료 후 retention 검증이 진짜 과제.

결과

프로모션 종료 후에도 일평균 콘텐츠 이용 120건 유지. 유입이 아닌 잔류 검증 완료.

03

AI 챗봇 단일 장애 위험 → GPT-4 + ML 이중 구조

상황

자살 위험 감지와 상담 챗봇을 하나의 AI 모듈로 처리하면 장애 시 전체가 멈추는 리스크 존재.

분석

GPT-4(위험 감지)와 ML 상담 챗봇을 독립 운영. 헬스케어 서비스에서 단일 장애는 허용 불가 — 두 모듈이 서로 의존하지 않도록 설계.

결과

상호 독립 구조로 장애 전파 차단. 이후 AI PoC를 더 초기 단계에 했어야 했다는 Lesson Learned로 남김.

Lesson Learned

✓ 유지해야 할 것

초기 타겟 정의는 가설 — 데이터 보고 빠르게 피벗하는 것이 PM의 기본
모객 전략과 retention 전략을 분리해서 설계 — 유입과 잔류는 다른 문제
헬스케어 AI는 단일 장애 없는 독립 구조로 — 리스크 설계가 기능 설계만큼 중요
재검사 주기를 지표별로 분리 — 우울/불안 2주, 스트레스 4주로 독립 검증

⚠ 보완해야 할 것

AI 챗봇 PoC를 기획 초기에 더 빨리 했어야 했다 — 후반에 검증하면 리스크가 커진다
간부 타겟 전환 시 콘텐츠 생산 체계도 함께 바꿨어야 했다
프로모션 의존 모객의 리스크를 초기부터 명시하고 retention KPI를 선행 설정했어야
"기획 단계의 타겟 정의는 가설이다. 데이터가 다른 말을 하면 바꿔야 한다."

사병을 위해 만들었는데, 간부가 더 많이 썼다.
그게 틀린 게 아니다 — 가설이 검증된 것이다.

군 정신건강 서비스를 만들면서 배운 건 하나다. 헬스케어 서비스에서 "효과가 있다"는 걸 증명하는 구조를 기획 단계부터 설계해야 한다는 것. 재검사 주기를 분리하고, 활성 유저의 개선 체감을 측정하고, retention을 프로모션과 분리해서 봐야 한다.
