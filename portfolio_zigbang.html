프로젝트 개요

배경

MAU 600만 부동산 플랫폼 직방에서 월세납부 서비스 PO. 기존 카드 결제만 지원하던 서비스에 오픈뱅킹(계좌이체)을 추가하고, 2주 스프린트로 funnel 분석 → UX 개선 → 배포 → 검증 사이클을 반복 운영했다. 정책 제약을 기술이 아닌 구조로 우회한 경험.

핵심 미션

오픈뱅킹 1회 200만원 한도를 3회 분할납부 정책으로 우회해 최대 600만원까지 가능하게 설계. CX 문의 데이터에서 아파트 세그먼트를 발굴해 고단가 사용자 유입으로 GMV 2억/월 달성. Tableau 기반 funnel 분석으로 전환율 +0.23% 누적 개선.

CX 데이터가 드러낸 숨은 세그먼트
+35%
아파트 세그먼트 타겟 전환 후 신규 유입 증가율

CX 문의에 "한도 올려달라"는 말이 반복됐다. 원룸·오피스텔 사용자가 200만원 한도에 막힐 이유가 없었다. Tableau로 한도 문의 유저의 유입 경로를 추적하니 아파트 관리비 영역에서 넘어오는 흐름이 보였다. 타겟이 아니었던 아파트 세그먼트가 GMV 성장의 핵심 레버였다.

데이터는 항상 말하고 있다. 듣고 있는가의 차이다.
CX 문의를 단순 CS로 처리했다면 아파트 세그먼트를 놓쳤을 것이다. 반복되는 불만이 곧 발굴되지 않은 수요였다.

이 결정이 아파트 세그먼트 확장과 맞물려 GMV 2억/월로 이어졌다.
문제 해결
01

오픈뱅킹 한도 제약 → 분할납부 정책으로 우회

상황

오픈뱅킹 정책상 1회 200만원 한도. 아파트 월세는 이를 초과하는 경우가 많아 고단가 사용자 유입 불가.

분석

정책은 바꿀 수 없고 기술 개발 시간도 없었다. 2주 안에 적용 가능한 방법은 정책 구조로 우회하는 것 — 3회 분할납부 설계.

결과

최대 600만원까지 납부 가능. 아파트 고단가 세그먼트 유입 가능해져 GMV 성장의 핵심 레버가 됐다.

02

CX 데이터 → 아파트 세그먼트 발굴 → 전략 전환

상황

서비스는 오피스텔 중심으로 설계되어 있었다. 한도 문의가 CX에 반복적으로 들어오고 있었지만 원인 불명.

분석

Tableau로 한도 문의 유저의 유입 경로 추적 → 아파트 관리비 영역에서 넘어오는 흐름 발견. 리드와 논의 후 아파트 타겟으로 전환.

결과

신규 유저 +35%, 고단가 납부 증가로 GMV 2억/월 달성. CX 데이터 → 분석 → 전략 전환까지 직접 주도.

03

funnel 분석 → 2주 스프린트 반복 → +0.23% 전환율

상황

월세납부 신청완료율이 약 1% 내외. 금융 결제 funnel 특성상 이탈 구간 파악이 핵심이었다.

분석

Tableau funnel 분석 → 이탈 구간 식별 → UX/카피 개선 → 배포 → 재분석. 2주 스프린트 4회 반복으로 11단계 → 9단계로 축소.

결과

+0.23% 전환율 향상. 금융 결제에서 0.1%도 GMV 단위로 유의미한 변화로 연결됐다는 걸 이때 체감했다.

Lesson Learned

✓ 유지해야 할 것

CX 데이터를 매일 보는 습관 — 반복 문의가 전략 전환의 시작이었다
기술 제약을 만나면 정책·구조로 우회하는 발상 — PM의 무기는 코드가 아니다
2주 스프린트 반복 — 작은 개선을 빠르게 쌓는 게 큰 숫자를 만든다
Tableau funnel 분석 → 이탈 구간 → UX 개선 → 재분석 사이클

⚠ 보완해야 할 것

아파트 세그먼트를 더 일찍 발굴했다면 — CX 데이터를 더 일찍 구조적으로 봤어야 했다
분할납부 정책의 사용자 혼란 가능성을 미리 검증했어야 했다
GMV 달성 후 다음 목표(재납부율, LTV)로 빠르게 전환했어야 했다
"가장 잘한 기획 결정은 오픈뱅킹 한도를 횟수로 우회한 것"

기술도 못 바꾸고 정책도 못 바꾸는 상황에서 2주 안에 풀어야 했다.
막히면 돌아가는 길을 찾는 게 기획자의 일이라는 걸 그때 다시 확인했다.

그리고 CX 데이터가 전략을 바꿨다. 데이터는 항상 말하고 있다. 듣고 있는가의 차이다.
